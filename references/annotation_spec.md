# 公文格式标注钩子协议（v3.0+）

> 本协议适用于 `official-document-formatter-hosted` 及所有基于 HTML 注释钩子的版本。
> 钩子格式：`<!--HOOK:TYPE-->段落内容`

## 钩子格式

每个段落必须以 HTML 注释形式的钩子开头，紧跟段落内容，**钩子与内容之间不加空格**：

```markdown
<!--HOOK:TITLE-->关于实施乡村振兴战略的意见
<!--HOOK:BODY-->为深入贯彻落实党中央、国务院决策部署……
<!--HOOK:H1-->一、总体要求
<!--HOOK:H2-->（一）指导思想
<!--HOOK:BODY-->坚持以人民为中心的发展思想……
```

## 支持的钩子类型

| 钩子 | 示例 | 说明 |
|------|------|------|
| `<!--HOOK:TITLE-->` | 关于××工作的报告 | 公文主标题，通常位于文档最开头，居中 |
| `<!--HOOK:H1-->` | 一、总体要求 | 一级标题（汉字数字 + 顿号），黑体 |
| `<!--HOOK:H2-->` | （一）指导思想 | 二级标题（括号汉字数字），楷体 |
| `<!--HOOK:H3-->` | 1．组织领导 | 三级标题（阿拉伯数字 + 全角点），仿宋加粗 |
| `<!--HOOK:H4-->` | （1）具体措施 | 四级标题（括号阿拉伯数字），仿宋加粗 |
| `<!--HOOK:BODY-->` | 为深入贯彻落实…… | 普通正文段落，首行缩进 2 字符 |
| `<!--HOOK:ATTACHMENT-->` | 附件：1.××方案 | 附件说明，格式与正文一致 |
| `<!--HOOK:SIGNATURE-->` | ××市人民政府 | 发文机关署名，右对齐 |
| `<!--HOOK:DATE-->` | 2025年3月15日 | 成文日期，右对齐 |

## 使用规则

1. **位置固定**：钩子必须位于段落最开头，中间不能有空格或换行。
2. **单一钩子**：一个段落只能有一个钩子。
3. **排版器逻辑**：`docx_formatter.py` 运行时会解析钩子类型，应用对应的国标格式后删除钩子文本。
4. **可人工修改**：用户可以在 `structured.md` 中直接增删段落、修改钩子类型或内容，再次运行 `--structured` 即可重新排版。
5. **无钩子兜底**：若某段落没有钩子，排版器会自动视为 `<!--HOOK:BODY-->`。

## CRITICAL — 编号→钩子映射规则（必须严格遵守）

**这是 AI 标注最容易出错的地方！必须根据编号格式判断钩子类型，而不是凭感觉。**

### 映射表

| 编号格式 | 正则 | 钩子 | 字体 |
|---------|------|------|------|
| 一、二、三、…… | `^[一二三四五六七八九十]+、` | `<!--HOOK:H1-->` | 黑体 |
| （一）（二）（三）…… | `^（[一二三四五六七八九十]+）` | `<!--HOOK:H2-->` | 楷体 |
| 1．2．3．…… | `^\d+．` | `<!--HOOK:H3-->` | 仿宋加粗 |
| （1）（2）（3）…… | `^（\d+）` | `<!--HOOK:H4-->` | 仿宋加粗 |

### ❌ 常见错误（绝对禁止）

```markdown
<!--HOOK:H2-->一、放假时间安排        ❌ "一、"是 H1，不是 H2！
<!--HOOK:H3-->（一）人身安全注意事项     ❌ "（一）"是 H2，不是 H3！
<!--HOOK:H4-->1．组织领导              ❌ "1．"是 H3，不是 H4！
```

### ✅ 正确标注

```markdown
<!--HOOK:H1-->一、放假时间安排          ✅ "一、" → H1（黑体）
<!--HOOK:H2-->（一）人身安全注意事项     ✅ "（一）" → H2（楷体）
<!--HOOK:H3-->1．组织领导              ✅ "1．" → H3（仿宋加粗）
<!--HOOK:H4-->（1）具体措施            ✅ "（1）" → H4（仿宋加粗）
```

### 判断口诀

> **看编号格式，不看内容层级！** "一、"永远是 H1，"（一）"永远是 H2，不管原文是否只有两级标题。
> 即使公文中没有"1．"级别的标题，"一、"和"（一）"也分别对应 H1 和 H2，绝不能因为"只有两级"就把它们标成 H2 和 H3。

## CRITICAL — 特殊规则（必须遵守）

### 1. 二级标题拆分
`<!--HOOK:H2-->` **只包含小标题本身**。如果原文中二级标题后面还跟着正文内容（如 "（一）指导思想。坚持以人民为中心……"），必须拆分为两个独立段落：

```markdown
<!--HOOK:H2-->（一）指导思想。
<!--HOOK:BODY-->坚持以人民为中心的发展思想……
```

### 2. 附件格式
`<!--HOOK:ATTACHMENT-->` 的格式与正文一致（仿宋三号），只是内容以"附件："开头，不需要额外加粗或变字体。

### 3. 落款与日期
落款单位和日期必须分别独立成段，且通常右对齐：

```markdown
<!--HOOK:SIGNATURE-->××市人民政府
<!--HOOK:DATE-->2025年3月15日
```

### 4. 标题独立成段
公文主标题必须单独一段，使用 `<!--HOOK:TITLE-->`，不要与正文混在同一段。
